Site Report

Introduction

The site was created as a web development assignment in the first TCA. The overall objective of the project was to create and develop a full-fledged, multi-page website using only HTML and CSS. The site serves as a personal portfolio and demonstrates my knowledge of basic web development principles such as semantic HTML, layout design, responsive design, and version control using GitHub.

The website includes all the required pages mentioned in the assignment brief: Home, Projects, Contact, Video Demo, and Site Report. Each page has a clear purpose and follows a consistent visual design. This project helped me understand how professional websites are structured and how design and usability play an important role in web development.

Planning and Structure

Before starting the development, I carefully reviewed the assignment requirements and planned the overall structure of the website. I decided to keep the navigation consistent across all pages so users could move easily between sections.

The general structure of each page follows this layout:

  • Header with navigation
  • Main content area
  • Footer

Semantic HTML elements such as <header>, <nav>, <main>, <section>, <article>, and <footer> were used to improve accessibility, readability, and maintainability. This follows the best practices taught during the module.

The Home page introduces me and the purpose of the website. The Projects page lists my areas of interest and includes project templates. The Video Demo page contains an embedded video demonstration of the website, while the Site Report page documents the development process and reflections.

Design Choices

The website was designed to be clean, modern, and professional. I aimed to keep the layout spacious and easy to read rather than crowded. The Home page features a large hero section with a background image to immediately attract the user’s attention.

The tagline “We Take and We Make” was kept simple and visually emphasized to match the overall theme of the site. Consistent typography, spacing, and colours were used across all pages. A green accent colour highlights buttons, links, and interactive elements to give the site a fresh and modern appearance.

Transparent, glass-style cards were used in sections such as Projects and Reviews to add visual depth while maintaining a minimal design. Hover effects were implemented using CSS only, enhancing user interaction without relying on animations or JavaScript.

Responsive Design

Responsiveness was a key focus of this project. The website is designed to work well on desktop, tablet, and mobile devices. CSS media queries, Grid, and Flexbox were used to ensure flexible layouts.

On larger screens, content is displayed in multiple columns, while on smaller screens it automatically stacks into a single column. This ensures readability and prevents layout overflow.

A CSS-only mobile navigation menu was created using a checkbox and label technique. On smaller screens, navigation links are hidden and replaced with a Menu button. When clicked, the menu expands vertically. This solution avoids JavaScript while still meeting assignment requirements.

Projects Section

The Projects page consists of two main parts. The first section lists my interests, including artificial intelligence, machine learning, database management, and networking, using a simple grid layout.

The second section displays four project templates in a scrollable layout. These templates are transparent, use a green theme, and can be scrolled or swiped on mobile devices. This adds interactivity while using only HTML and CSS. On larger screens, the cards adjust to fill the available space, providing a good user experience.

Reviews / Testimonials Section

A Reviews section was added to the Home page to make the website feel more realistic and professional. This section includes testimonial cards written as feedback for a web developer or website designer.

Each card contains a short review, name, role, and star rating. The reviews are displayed in a responsive grid layout: three columns on desktop, two on tablets, and one on mobile devices. This section improves the visual appeal and demonstrates structured content layout design.

Contact Page

The Contact page includes a simple form that uses the mailto method. When the form is submitted, the user’s default email application opens with the form data included.

This approach was chosen because server-side scripting is outside the scope of this module. The form uses proper labels and input fields to ensure accessibility and usability, and the design remains consistent with the rest of the website.

Video Demonstration

The Video Demo page contains an embedded video using an <iframe>. The video demonstrates the website’s navigation, features, and responsiveness. Embedding the video directly allows it to play within the site instead of redirecting users to an external platform.

Validation and Testing

All HTML and CSS files were validated using online validation tools to ensure there were no major syntax errors. Minor warnings were reviewed and corrected where necessary. Validation screenshots are included in this Site Report as required.

The website was tested on different screen sizes using browser developer tools. Navigation links, forms, and interactive elements were checked to ensure they function correctly.

Version Control and GitHub

GitHub was used throughout the development process for version control. Regular commits were made with clear messages describing updates such as layout improvements, responsiveness fixes, and content additions. This demonstrates good development practice and provides evidence of continuous progress.
